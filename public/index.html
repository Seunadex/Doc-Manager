    Introduction

    Welcome to Document Management API! You can use our API to access various API endpoints, which can give you information on how to register, login, add a new document and manage your document in our database.

    This example API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API's documentation.

    Authentication

    The API uses traditional means of authentication (username and password) to allow access to the API. Kindly see the Create a New User section under Users in the API documentation. A token is generated on registering or login.

    It is expected that the token is to be included in all API requests to the server in a header that looks like the following:

    Authorization: token

    To get access to the document, you have to be registered: Register and get a token to be inserted in the header

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files...just save them to the `includes` folder and add them to the top of your `index.md`'s frontmatter. Files are included in the order listed.

    The Document manager API uses the following error codes:

    Error Code Meaning
    200 OK
    201 Created
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- wrong username/password.
    403 Forbidden -- Access denied.
    409 Conflict -- Request already exist
    404 Not Found -- The specified request not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
